# cc-switchy v0.4.0 Grok and TUI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Support CC Switch v3.18/v16 Grok Build data and redesign the TUI around Agent provider switching and remote synchronization, then ship v0.4.0.

**Architecture:** Extend the existing `Agent` abstraction with an exclusive Grok Build target whose live state is `~/.grok/config.toml`, using the current transactional provider-switch and projection pipeline. Keep sync protocol db-v6 unchanged, repair the new v15 columns during import/restore, read absent Grok flags as disabled on ordinary legacy-database opens, and reorganize Ratatui rendering with only a Unicode-width helper added to the existing UI stack.

**Tech Stack:** Rust 2021, Rusqlite, Serde JSON, toml/toml_edit, Ratatui, Crossterm, Tokio.

## Global Constraints

- Preserve unrelated user work and existing rollback behavior.
- Keep the remote protocol at `version = 2` and `dbCompatVersion = 6`, matching CC Switch v3.18.
- Do not port CC Switch proxy, OAuth, session-usage, or desktop frontend code.
- Grok Build is an exclusive Agent with provider, MCP, and Skills projection.
- Release version is `0.4.0`; tag is `v0.4.0`.
- The TUI follows `docs/design/terminal-design-dna.json`.
